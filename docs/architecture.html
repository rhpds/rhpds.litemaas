Component Reference

LiteMaaS Application (litellm-rhpds namespace)

ComponentImageReplicasResourcesPurpose
litellm-frontend quay.io/rh-aiservices-bu/litemaas-frontend:0.4.0 3 100m CPU / 128Mi–256Mi RAM React SPA served via nginx. User-facing UI for model browsing, API key management, usage dashboard.
litellm-backend quay.io/rh-aiservices-bu/litemaas-backend:0.4.0 3 200m CPU / 512Mi–1Gi RAM Fastify/Node.js API. Manages users, subscriptions, API keys, model sync, usage analytics, branding.
litellm quay.io/rh-aiservices-bu/litellm-non-root:main-v1.81.0-stable-custom 3 500m CPU / 1Gi–4Gi RAM LiteLLM proxy. Routes inference requests to model backends. Custom fork adds /v1/tokenize and /docling endpoints.
litellm-postgres postgres:16-alpine 1 — PostgreSQL 16. Hosts both litemaas_db (LiteMaaS tables) and litellm_db (LiteLLM tables) on the same instance.
litellm-redis registry.redhat.io/rhel9/redis-7:latest 1 — Redis 7. LiteLLM uses it to cache model list and key lookups. Backend flushes cache after model CRUD so all proxy pods pick up changes immediately.

Init Containers

PodInit ContainerWhat it does
litellm-frontend copy-custom-logos Copies RHDP-branded SVG logos into a shared volume. Patches JS bundle to replace default LiteMaaS text with RHDP branding. Nginx then serves from this volume.
litellm-backend wait-for-database Polls PostgreSQL until it accepts connections before starting the backend.
litellm-backend run-migrations Runs LiteMaaS DB schema migrations on startup. Ensures litemaas_db schema is up to date.

Automation (Bastion Host)

ScriptScheduleWhat it does
cleanup-litemaas-keys-litellm-rhpds.sh Daily at 2 AM Fetches all virtual keys from LiteLLM. Deletes expired keys and keys older than 30 days. After each deletion, marks the corresponding api_keys record in LiteMaaS as inactive. Final pass cleans any remaining orphaned LiteMaaS keys.

Key Lifecycle — Creation & Cleanup

sequenceDiagram autonumber actor A as 🔧 Admin participant BE as ⚙️ LiteMaaS Backend participant LLM as 🔀 LiteLLM participant PG_LM as 🗃 litemaas_db participant PG_LL as 🗃 litellm_db participant CR as ⏰ Bastion Cronjob
(2AM daily) A->>BE: Create API key for user
(models, TPM, RPM, budget) BE->>LLM: POST /key/generate
with limits LLM->>PG_LL: INSERT LiteLLM_VerificationToken LLM-->>BE: sk-... key BE->>PG_LM: INSERT api_keys
(litellm_key_alias, key_hash) BE-->>A: show key once note over CR: After 30 days or expiry CR->>LLM: GET /key/list (paginated) CR->>LLM: POST /key/delete (expired keys) LLM->>PG_LL: DELETE from VerificationToken CR->>PG_LM: UPDATE api_keys SET is_active=false
WHERE litellm_key_alias=...
📌 External services (WatsonX, Bedrock) are planned but not yet in production. When enabled, LiteLLM will route requests to them using the same virtual key authentication — users will not need to change anything.
